## Onboarding goals
- This is not a check-in app
- One sentence is enough
- The system gradually learns patterns
- Follow-ups are occasional, not constant

## Status flows
- Pattern: candidate -> observing -> active -> fading -> archived
- Friction: candidate -> observing -> active -> resolved -> archived
- Opportunity: candidate -> observing -> emerging -> pilot_ready -> build_ready/testing -> adopted/rejected
